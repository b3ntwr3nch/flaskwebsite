Robotics & Microcontroller: Self-Driving Signal-Following Car

Project Overview

This project aims to create a self-driving car that automatically drives and follows signals. It combines robotics, microcontrollers, and software-defined radio (SDR) technology to create an intelligent, autonomous vehicle capable of navigating based on received signals.

Key Components

  • Brain: Raspberry Pi - Acts as the central processing unit
  • Motion Sensing: MPU-6050 - Provides gyroscope and accelerometer functionality
  • Motor Control: DRV8833 Dual DC Motor Driver - Controls the car's motors
  • Signal Processing: Software-Defined Radio (SDR) - For automatic signal following

MPU-6050 Sensor for Arduino Raspberry Pi

The MPU-6050 is crucial for our self-driving car, providing motion sensing capabilities:

  • Power Supply: 3V-5V
  • Gyroscope Range: ±250, ±500, ±1000, ±2000°/s
  • Acceleration Range: ±2g, ±4g, ±8g, ±16g
  • Communication: Standard IIC communication protocol
  • Built-in 16-bit AD converter, 16 bits of data output
  • Pin spacing: 2.54mm

DRV8833 Dual DC Gear Motor Driver Module

The DRV8833 is essential for controlling the car's movement, replacing TB6612FNG:

  • Input Voltage: 2.7V-10.8V
  • Single H-bridge Output Current: 1.5A, can control 2 DC aligned motors
  • Built-in protection: Overcurrent, short-circuit, under-voltage lockout, and overheating
  • Low-power sleep mode
  • Can drive DC motors under 1.5A and 4-wire stepper motors

Pin Description:

  • AIN1, AIN2: Logic input control ports for AO1 and AO2, level 0-5V
  • BIN1, BIN2: Logic input control ports for BO1 and BO2, level 0-5V
  • AO1, AO2, BO1, BO2: H-bridge output ports for connecting DC motors
  • GND: Ground
  • VM: Chip and motor power supply pin, voltage range 2.7V-10.8V
  • STBY: Standby control, connect to 5V to work; level 0-5V

SDR Integration for Signal Following

The project incorporates Software-Defined Radio (SDR) technology to enable the car to follow signals automatically. This allows the vehicle to:

  • Detect and analyze various radio signals in real-time
  • Adjust its path based on the strength and direction of received signals
  • Potentially communicate with other devices or infrastructure

Self-Driving Capabilities

By combining the MPU-6050 for motion sensing, the DRV8833 for motor control, and SDR for signal processing, our car achieves self-driving capabilities including:

  • Autonomous navigation based on sensor data and received signals
  • Obstacle detection and avoidance
  • Dynamic path planning and adjustment
  • Signal strength-based movement and decision making

Future Enhancements

Potential improvements and additions to the project could include:

  • Integration of machine learning algorithms for improved decision making
  • Addition of cameras for visual navigation and object recognition
  • Expansion of SDR capabilities for broader signal analysis and response
  • Implementation of swarm behavior with multiple vehicles
